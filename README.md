# Библиотека для работы с матрицами реализованная с помощью типа данных mpz_t

## Структура проекта

```
3_mpz_t/
├── include/
│   ├── Fraction.h
│   └── Matrix.h
├── src/
│   ├── Fraction.cpp
│   └── Matrix.cpp
├── test/
│   ├── test_matrix.cpp
│   ├── mpq_tDATA1.txt
│   └── mpq_tDATA2.txt
├── lib/
│   └── .gitkeep
├── Makefile
├── README.md
└── .gitignore
```


1. **include** — содержит заголовочные файлы `Fraction.h` и `Matrix.h`, в которых описаны классы `Fraction` и `Matrix`.

2. **src** — содержит файлы `Fraction.cpp` и `Matrix.cpp`, в которых реализованы методы классов `Fraction` и `Matrix`.

3. **test** — содержит файлы с входными данными `mpq_tDATA1.txt` и `mpq_tDATA2.txt`, а также файл `test_matrix.cpp`, демонстрирующий работу с библиотекой.

**Реализованы следующие алгоритмы:**

* умножения матрицы на вектор;
* классический алгоритм умножения матриц;
* блочный алгоритм умножения матриц;
* алгоритм Штрассена для умножения матриц;
* решение систем линейных алгебраических уравнений методом Гаусса;
* решение систем линейных алгебраических уравнений методом Жордана-Гаусса;
* нахождение обратной матрицы методом Гаусса;
* нахождение обратной матрицы методом Жордана-Гаусса.

## Makefile

Для автоматизации сборки библиотеки и тестовой программы используется `Makefile`.

Поддерживаются следующие команды:

- `make` или `make all` — сборка статической и динамической библиотек.
- `make test_static` — сборка тестовой программы со статической библиотекой.
- `make test_dynamic` — сборка тестовой программы с динамической библиотекой.
- `make test_all` — сборка тестовой программы с статической и с динамической библиотеками.
- `make clean` — удаление объектных файлов, библиотек и собранных тестовых программ.
- `make run_static` — запустить тестовую программу со статической библиотекой;
- `make run_dynamic` — запустить тестовую программу с динамической библиотекой;

## Запуск тестовой программы:

```
make all

make test_all

make run_static
или  
make run_dynamic
```
